6. Authentication Flow

Authentication flow нь register, OTP verify, login, token usage, refresh, logout гэсэн үндсэн lifecycle-тэй.

Register -> OTP Verify -> Login -> Access Token -> Protected Route -> Refresh -> Logout
  1. User register хийнэ.
  2. Server password-г bcrypt ашиглан hash хийнэ.
  3. Server 6 оронтой OTP үүсгэж hash хэлбэрээр хадгална.
  4. User OTP verify хийсний дараа isVerified=true болно.
  5. User email/phone болон password ашиглан login хийнэ.
  6. Server access token болон refresh token буцаана.
  7. Protected route дээр Authorization: Bearer <accessToken> ашиглана.
  8. Access token expire бол refresh token ашиглан шинэ token авна.
  9. Refresh үед refresh token rotate хийгдэж хуучин token хүчингүй болно.
  10. Logout үед refresh token database-аас устгагдана.

Login Request Example

{
  "identifier": "donor@example.com",
  "password": "StrongPass123"
}

Login Response Example

{
  "message": "Login successful",
  "accessToken": "jwt_access_token",
  "refreshToken": "jwt_refresh_token",
  "user": {
    "email": "donor@example.com",
    "role": "donor",
    "isVerified": true
  }
}

7. OTP Verification Flow

OTP verification нь бүртгүүлсэн хэрэглэгчийн email/phone эзэмшлийг баталгаажуулах зорилготой. OTP нь 6 оронтой, 2 минутын хүчинтэй хугацаатай, 3 удаа буруу оруулахад lock болно.

OTP үүсэх -> OTP hash хадгалах -> 2 минут TTL -> Verify -> Success эсвэл Lock
Policy Утга Зорилго
OTP length 6 digit Хэрэглэгчид оруулахад энгийн, brute-force хамгаалалттай хосолно
Expiration 2 минут Хуучин OTP ашиглах эрсдэлийг багасгана
Failed attempts 3 удаа OTP таах оролдлогыг хязгаарлана
Storage bcrypt hash OTP raw value database-д хадгалагдахгүй

OTP Verify Request Example

{
  "identifier": "donor@example.com",
  "otp": "123456"
}

9. Security Tactics

Модуль нь authentication болон authorization хэсэгт шаардлагатай хамгаалалтын тактикуудыг backend түвшинд хэрэгжүүлсэн.

Security tactic Хэрэгжүүлэлт Хамгаалж буй эрсдэл
bcrypt password hashing Password-г bcrypt.hash ашиглан хадгална Plain text password leak
JWT access token sub, role payload бүхий signed token Unauthorized protected route access
Refresh token rotation Refresh бүр дээр шинэ token үүсгэж хуучныг invalid болгоно Refresh token reuse
Refresh token invalidation Logout үед refresh token hash database-аас устгана Logout дараах session reuse
OTP expiration OTP 2 минут хүчинтэй Хуучин OTP ашиглах
OTP attempt lock 3 буруу оролдлогын дараа lock OTP brute-force
Input validation express-validator ашигласан Invalid input, bad request
Rate limiting Login, OTP, refresh endpoint дээр limit тавьсан Brute-force болон abuse
Helmet helmet() global middleware Security header дутуу байх
CORS CORS_ORIGIN env тохиргоо Untrusted origin access
Sensitive fields болох password, otp.codeHash, refreshTokens нь Mongoose schema дээр default query-д сонгогдохгүй байхаар select: false тохируулагдсан.
